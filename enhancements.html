Implement Responsive Design - Hamburger Menu

On all pages, the navigation bar on both the top and bottom of the page will change from a button menu to a drop down, hamburger menu when the screen size is less than 600px
This prevents the navigation buttons from squishing too small or overlapping eachother and leaving the container
This is done using a checkbox hidden behind 3 div tags with borders to make the shape of a hamburger menu
When the screen is smaller than 680px, the navigation buttons will be hidden and the hamburger menu will be displayed
You can see an example of this in the nav menus of all pages or on this page when the screen is below 680px here.

The following references were used:

  • Create a Responsive Hamburger Menu with HTML and CSS: A step-by-step tutorial - Mukund Kumra for medium.com
  • How To - Responsive Sidebar - W3schools

CSS Variables — var() function

CSS variables were used to define a universal colour scheme that can be easily referenced to streamline the process of applying colours to elements, and maintain consistency throughout the project.

Checkbox Buttons

The checkbox hack was used in the jobs page to act as a method of controlling other HTML elements to 'expand' content related to the job positions. The checkbox is used in conjunction with a connected label, and when the label is clicked by users, a non-displayed div element's css display element is changed to 'block' to reveal its content.
You can see an example of this here.
